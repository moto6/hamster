## 포트 in/out 방향 햇갈릴때
헥사고날 아키텍처에서 포트의 방향은 **"제어의 흐름(Flow of Control)"**이 아니라, **"누가 이 인터페이스를 필요로 하느냐"**와 **"의존성의 방향"**에 결정됩니다.

비즈니스 로직이 외부 세계(라이브러리, DB, 외부 API)에 무언가를 요청하기 위해 사용하는 인터페이스는 모두 Out Port입니다.

In Port (Primary)	

역할	: 외부에서 내부(Domain)로 들어오는 관문
호출하는곳() : 어댑터 (Controller 등)
구현체 :비즈니스 로직 (Service)

Out Port (Secondary)
역할 : 내부(Domain)에서 외부로 나가는 통로
호출하는곳 : 비즈니스 로직 (Service)
구현체 : 어댑터 (Persistence, JWT 등)


### 한줄요약
- InPort :  제공해주는 기능들 
- OutPort : 외부에 요청할때 // 이 파일이 의존하는 다른 계약

## 백엔드 관련
Thread vs Coroutine: 물리적 스레드와 논리적 루틴의 차이 (경량 스레드 개념)
Suspending vs Blocking: 스레드를 점유하며 기다리는가, 스레드를 양보하며 멈추는가


The suspend Keyword: 컴파일러가 비동기 콜백 체인을 순차적 코드로 변환하는 마법의 시작점
Coroutine Builders (launch, async): 결과를 반환하지 않는 작업과 결과를 기다리는 작업의 구분
Structured Concurrency: 부모와 자식 코루틴의 생명주기를 묶어 메모리 누수를 방지하는 구조


Dispatchers (IO, Default, Main): I/O 작업과 CPU 작업에 따라 적절한 스레드 풀을 배정하는 법

withContext: 코드 실행 중에 스레드 환경(Context)을 안전하고 가볍게 전환하는 도구

Reactor Interop (awaitSingle): Mono/Flux를 기다려(Await) 결과값만 쏙 빼오는 브릿지 기법

Flow (Cold Stream): Flux에 대응하는 코틀린 표준의 비동기 데이터 스트림 처리

Sequential by Default: 복잡한 비동기 로직을 일반 if, for, try-catch로 작성하는 패러다임
