# 요구사항

## 백엔드 시스템 디자인

### 기술적 요구사항
- JDK : 최소 JDK 21 이상 사용
- Lang : Java or Kotlin 사용
- Framework : 최소 Spring Boot 4.0 이상 버전.
    - Non-Blocking 기반 코드를 작성해주세요 (예시 : R2DBC, ReacviteWeb, WebFlux, Coroutines 등...)
- DB : 관계형 데이터베이스를 사용해주세요 (PostgresQL, MySQL, H2 중 택1 )

### 기능적 요구 사항
- 도메인 모델링: Book, Rating, User 엔티티 정의 및 관계 설계
- 평점 등록 API: 사용자가 도서에 평점을 부여하는 기능
- 실시간 평점 조회 API: 특정 도서의 현재 평점을 확인하는 기능
- 최근 1시간 통계 API: 1시간 내 평점 등록 수가 가장 많이 증가한 Top 10 도서 집계
- 핫 랭킹 추천 API: 평점 수와 점수를 결합한 본인만의 알고리즘을 기반으로 도서 정렬 및 추천
- A/B 테스트: 핫 랭킹 정책 변경 및 비교를 위한 테스트 구조 구현

### 비 기능적 요구사항
- 데이터 처리 및 성능 최적화
    - 고효율 데이터 인계 : 대용량 기초 데이터 파일을 효율적으로 관계형 데이터베이스(RDB)에 적재해주세요
    - 인덱싱 전략 : 데이터 성능을 극대화하기 위해 최적화된 DB 인덱싱을 사용해주세요
    - 메모리 관리 : 대용량 처리 시 Memory 부하를 방지하기 위해 다양한 기법을 활용하여 메모리 점유율을 최소화해주세요
- 개발 프로세스 및 품질
    - 테스트 코드 : 충분한 테스트 코드를 작성해주세요
    - 기능별 PR : 개발 단계별/기능별로 분리하여 Pull Request를 작성해주세요
    - 문서화 : 모든 설계와 안내는 Markdown 기반으로 문서화해주세요
- 개발 원칙 및 책임
    - 코드 오너십 : AI의 도움을 받더라도 본인이 100% 이해하고 유지보수할 수 있는 코드만 반영하세요.

### 이외 잡스러운 개인목표
- 인프런에 백엔드 개발자 과제테스트 준비하기 핵심편

### 기타사항
- 데모용 프론트엔드 페이지 작성이 가능합니다. (점수에 영향 없음)
- 관계형 데이터베이스만을 사용하는것을 권장합니다. 이외의 별도 인프라 요소 사용은 가능은 하나 권장하지는 않으며 꼭 필요한 경우에 한하여 사용합니다.
- 멀티테넌시 지원하는 도서관 관리자 시스템을 만들어야합니다
- 인프라에 의존적이지 않은 코드 : 인프라 변경이 쉽도록 rdbms 전문검색 에서 갈아끼우기 쉽도록 (ES등 다른 인프라로)
