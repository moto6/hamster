// BookInventoryMaster

-----
- 백엔드 구현사항은 신경쓰지마 무슨 API 구현해야하는지만 리스트로+DTO 로 API DOCS 로 정리해
- 도메인 모델링: BookSkuMaster, BookInventoryDetail LoanMaster, LoanDetail, Reservation, LoanOverdue, BookRating, UserMaster, 엔티티 정의 및 관계 설계


- (어드민 페이지) BookSkuMaster 관리페이지 만들어, 스큐 등록하는 기능도 만들고
- (어드민 페이지) 대출기록조회 페이지 LoanMaster, LoanDetail 도메인 기반으로 만들어, 유저이름 혹은 책 두가지 검색조건으로 날짜 지정해서 검색하도록 기본은 최근30일이고 시작일-종료일이 최대 180 일 이내에만 검색가능하도록
- (유저 페이지) 유저입장에서 나의 대출기록 확인 페이지 (최근 30일만 조회되고, 기간을 변경하면 오늘~최대 180 기록까지만 조회 가능하도록)
- (어드민 페이지) 어드민 입장에서 도서 예약내역 확인하는 페이지가 필요해. 예약시간 기준으로 보여야하고(기본 지난7일), 검색 필터로는 날짜별(최대 180일), 유저이메일, 도서제목, 도서ISBN 기준 검색필터 필요함
- (유저 페이지) 나의 예약도서 확인 페이지 (최근 30일만 조회되고, 기간을 변경하면 오늘~최대 180 기록까지만 조회 가능하도록), 예약은 "예약대기"/"예약대출가능"/"예약취소"/"대출됨" 상태 중 하나로 표시되어야함 (API 에서 제공해줌)
- (유저 페이지) 자료검색페이지 만들어, 키워드를 입력해서 검색하고 키워드 속성은 유저가 컨트롤할수 있어 (전체, 도서명, 저자)
    - 검색조건에 해당하는 책들이 도서명,저자, 출판사, 출판년도, 청구기호, 예약이 있는경우에만 예약 몇명인지, 반납예정일이 언제인지
    - 도서 상태의 경우 대출불가/대출가능 두가지이며 대출불가의 경우 상세내용(대출중,정비중) 보여지고, 도서대출 가능한 경우에는 "상호대차" 버튼이, 대출불가인 경우에는 "도서예약" 버튼이 보여야해
- (어드민 페이지) 대출 연체중인 정보가 표시되는 화면 만들어줘 책제목, ISBN, 청구기호, 연체자, 연체일수 등등 이 표시되어야 해
- (어드민 페이지) 등록된 도서 평점(+리뷰)을 보여주는 페이지, 평점과 코멘트/리뷰
  ㄴ 평점 등록 기능 : 사용자가 도서에 평점을 부여하는 기능
  ㄴ BookSkuMaster 에 있는 ISBN 코드 기준으로 도서평점 및 리뷰 남기는 기능 페이지
  ㄴ 평점등록기능을 API 호출하는 코드 만들기

- (유저 페이지) 실시간 평점 조회 : 특정 도서의 현재 평점을 확인하는 기능 만들기
- (유저 페이지) 최근 평점 등록 수가 가장 많이 증가한 책(실시간 랭킹 페이지)
  ㄴ 최근 10시간,1시간, 7일, 30일 단위 통계 Top 10 도서 집계한 API 가 주어질때 이를 표시하는 페이지
  ㄴ 동일한 API 에서 파라미터만 바꿔서 호출하는 구조임
-  (유저 페이지) 핫 랭킹 추천 페이지 : 평점 수와 점수를 결합한 알고리즘을 기반으로 도서 정렬 및 추천 페이지
   ㄴ 핫랭 킹 추천 API 를 호출해야하며 API 호출할때 성별(남,녀,미상), 연령,지역, 주제 정보를 같이 넘겨서 호출하면 백엔드에서 응답해줄꺼야
